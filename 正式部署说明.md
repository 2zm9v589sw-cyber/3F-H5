# 西宁城北吾悦广场暑期3楼特别活动 H5 云端正式版部署说明

## 一、正式版入口

部署完成后建议固定为以下入口：

- 商户发券端：`https://你的域名/merchant`
- 商户核销端：`https://你的域名/redeem`
- 后台管理端：`https://你的域名/admin`
- 顾客券面：`https://你的域名/coupon?code=券码`

顾客不需要固定入口，顾客通过商户生成的二维码进入自己的券面。

## 二、当前正式版已具备功能

- 商户发券
- 顾客扫码保存券面
- 商户扫码核销
- 后台查看核销数据
- 后台导出 CSV
- 后台修改活动名称
- 后台修改券面权益说明
- 后台修改满额赠券条件
- 后台新增/修改/停用商户
- 后台新增/修改/停用券类型
- 后台控制亲子畅玩券和品牌复购券的核销范围
- 后台密码登录保护
- 数据保存到 Supabase 云数据库

## 三、默认活动规则

默认活动时间：

- 2026-07-01 至 2026-08-31

默认满额赠券条件：

- 儿童零售消费满 299 元
- 运动户外消费满 599 元
- 体验类消费满 199 元

默认券面权益说明：

`当天有效，逾期自动作废；核销以后台券码状态为准。`

## 四、泄漏风险与控制方式

正式版不会把数据库密钥放在前端页面里，数据库密钥只保存在 Vercel 环境变量中。

权限控制方式：

- 顾客端只能查看自己的券码。
- 商户端只能发券和核销，不能进入后台配置。
- 后台端需要管理员密码登录。
- Supabase 数据库启用 Row Level Security，前端不能直接读取数据库。
- 数据库操作全部经过服务端 API。

仍需注意：

- 后台管理链接和密码不要发给商户。
- 管理员密码建议单独设置，不要使用常用个人密码。
- 如果商户人员离职或活动结束，应及时更换后台密码。

## 五、Supabase 数据库部署

1. 打开 Supabase，创建新项目。
2. 进入 SQL Editor。
3. 复制 `supabase/schema.sql` 全部内容。
4. 执行 SQL。
5. 进入 Project Settings → API。
6. 记录：
   - Project URL
   - service_role key

注意：`service_role key` 只能放在 Vercel 环境变量中，不能发到群里，也不能写进前端页面。

## 六、Vercel 部署

1. 打开 Vercel。
2. 新建 Project，导入本项目文件夹。
3. 设置环境变量：

```text
SUPABASE_URL=你的 Supabase Project URL
SUPABASE_SERVICE_ROLE_KEY=你的 Supabase service_role key
ADMIN_PASSWORD=你的后台管理密码
ADMIN_SESSION_SECRET=一串随机长密钥
NEXT_PUBLIC_APP_BASE_URL=https://你的域名或vercel域名
```

4. 部署。
5. 部署完成后打开 `/admin`，用 `ADMIN_PASSWORD` 登录后台。

## 七、后台日常修改方法

### 修改商户新权益

进入后台 `/admin`：

1. 找到“活动基础配置”。
2. 修改“券面权益说明”。
3. 点击“保存全部配置”。

保存后，新生成的券会使用新权益说明；历史已生成券保留生成时的说明。

### 新增商户

进入后台 `/admin`：

1. 找到“商户/点位配置”。
2. 点击“新增商户”。
3. 填写铺位号、商户名称、类别。
4. 设置是否为亲子多经点位、是否可发券、是否可核销。
5. 点击“保存全部配置”。

### 修改满额条件

进入后台 `/admin`：

1. 找到“满额赠券条件”。
2. 修改类别名称或满额金额。
3. 点击“保存全部配置”。

### 修改券类型和核销范围

进入后台 `/admin`：

1. 找到“券类型”。
2. 修改券类型名称或核销范围。
3. 亲子畅玩引导卡建议设置为“仅亲子多经点位”。
4. 品牌复购引导券建议设置为“正铺/主次力店”。
5. 点击“保存全部配置”。

## 八、活动结束后

活动结束后建议：

1. 导出核销 CSV。
2. 关闭商户发券权限，或修改活动结束日期。
3. 更换后台密码。
4. 保留数据库作为活动复盘资料。
